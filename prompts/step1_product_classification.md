# Step 1: 상품분류 추출

아래는 한화생명 보험 사업방법서를 마크다운으로 변환한 문서입니다.
이 문서에서 **모든 보험 상품의 분류 정보**를 추출하세요.

## 문서
```
{{DOCUMENT}}
```

## 추출 규칙

### 1. 상품명칭 추출
- "보험종목의 명칭" 섹션에서 상품명칭(대표 상품명)을 찾으세요.
- 보통 "한화생명 ○○보험 무배당" 형태입니다.

### 2. 세부종목 축(axis) 추출
- "보험종목의 구성" 또는 "세부보험종목" 테이블에서 세부종목 축을 추출하세요.
- 테이블의 각 열이 하나의 축입니다:
  - 첫 번째 열의 항목들 → 세부종목1 (예: 일반가입형, 간편가입형(2년))
  - 두 번째 열의 항목들 → 세부종목2 (예: 해약환급금 미지급형, 표준형)
  - 이런 식으로 세부종목10까지 가능
- 각 열의 항목이 콤마(,)로 구분되어 있으면 개별 항목으로 분리하세요.

### 3. 판매 제외 항목 처리
- "판매하지 않음", "판매 제외", "비교용" 등으로 명시된 항목은 **제외**하세요.
- 예: "'표준형'은 판매하지 않으며 비교용으로 사용함" → 표준형 제외

### 4. 모든 조합 생성
- 상품명칭 × 세부종목1 × 세부종목2 × ... 의 **모든 가능한 조합(cross-product)**을 생성하세요.
- 상품명 = 상품명칭 + " " + 세부종목1 + " " + 세부종목2 + ...

### 5. 종속특약 관련
- "종속특약"이 별도로 언급된 경우, 종속특약의 상품도 포함하세요.
- 단, 종속특약은 상품명칭이 다릅니다 (예: "한화생명 ○○특약 무배당").

## 출력 형식

JSON 배열로 출력하세요. 각 항목:

```json
[
  {
    "상품명칭": "한화생명 ○○보험 무배당",
    "세부종목1": "일반가입형",
    "세부종목2": "해약환급금 미지급형",
    "상품명": "한화생명 ○○보험 무배당 일반가입형 해약환급금 미지급형"
  }
]
```

- 세부종목이 없는 축은 필드를 포함하지 마세요.
- 모든 조합을 빠짐없이 나열하세요.
