# 사용 가이드: Claude 프로젝트 기반 세트데이터 추출

## 프로젝트 설정 (1회)

### 1. Claude 프로젝트 생성
1. [claude.ai](https://claude.ai) 접속
2. 좌측 사이드바 → "Projects" → "Create a project"
3. 프로젝트 이름: "한화생명 세트데이터 추출" (자유)

### 2. Custom Instructions 등록
1. 프로젝트 설정 → "Custom instructions"
2. `project_instructions.md` 파일의 **전체 내용**을 복사하여 붙여넣기
3. 저장

### 3. 프로젝트 파일 업로드
아래 2개 파일을 프로젝트에 업로드합니다:

| 파일 | 설명 |
|------|------|
| `product_overrides.json` | 상품별 override 규칙 (config/ 폴더에서 복사) |
| `보종코드_상품코드_매핑_utf8.csv` | 보종코드↔상품코드 매핑 테이블 (UTF-8) |

## 사용 방법

### 기본 사용
1. 프로젝트 내에서 새 대화 시작
2. DOCX 파일을 대화창에 **드래그앤드롭** 또는 **파일 첨부**
3. Claude가 자동으로 5개 데이터셋을 추출하여 아티팩트로 표시

### 대화 예시

```
사용자: [한화생명 H간병보험 무배당 사업방법서.docx 업로드]

Claude: 사업방법서를 분석합니다.

Phase 1 완료: 상품분류 3건 추출
Phase 2 완료: 납입주기 3건, 보험기간 20건, 연금개시나이 0건, 가입가능나이 20건
Phase 3 완료: override 적용 + 코드매핑

[아티팩트: 상품분류 결과 테이블]
[아티팩트: 납입주기 결과 테이블]
[아티팩트: 가입가능보기납기 결과 테이블]
[아티팩트: 가입가능나이 결과 테이블]
[아티팩트: 통합 JSON + CSV 다운로드]
```

### 추가 요청 예시

```
사용자: 납입주기 결과를 CSV로 내보내줘
사용자: 상품분류에서 치매보장플랜형이 빠진 것 같은데 확인해줘
사용자: 가입가능나이를 보험기간별로 정리해서 보여줘
```

## 출력 구조

각 데이터셋의 JSON 구조:

### 상품분류
```json
{"상품명칭": "한화생명 ○○보험 무배당", "세부종목1": "...", "상품명": "...",
 "isrn_kind_dtcd": "...", "isrn_kind_itcd": "...", "prod_dtcd": "...", "prod_itcd": "..."}
```

### 납입주기
```json
{"상품명": "...", "납입주기": [{"납입주기명": "월납", "납입주기값": "1", "납입주기구분코드": "M"}],
 "isrn_kind_dtcd": "...", ...}
```

### 가입가능보기납기
```json
{"상품명": "...", "가입가능보기납기": [
  {"보험기간": "X90", "보험기간구분코드": "X", "보험기간값": "90",
   "납입기간": "N5", "납입기간구분코드": "N", "납입기간값": "5",
   "최소제2보기개시나이": "", "최대제2보기개시나이": "", "제2보기개시나이구분코드": ""}
], "isrn_kind_dtcd": "...", ...}
```

### 보기개시나이 (연금상품만)
```json
{"상품명": "...", "보기개시나이정보": [
  {"제2보기개시나이": "X55", "제2보기개시나이구분코드": "X", "제2보기개시나이값": "55", "성별": "남자"}
], "isrn_kind_dtcd": "...", ...}
```

### 가입가능나이
```json
{"상품명": "...", "가입가능나이": [
  {"최소가입나이": "15", "최대가입나이": "80", "성별": "",
   "최소보험기간": "90", "최대보험기간": "90", "보험기간구분코드": "X",
   "최소납입기간": "5", "최대납입기간": "5", "납입기간구분코드": "N",
   "최소제2보기개시나이": "", "최대제2보기개시나이": "", "제2보기개시나이구분코드": ""}
], "isrn_kind_dtcd": "...", ...}
```

## 주의사항

- DOCX 파일만 지원합니다 (PDF 미지원).
- 한 번에 하나의 사업방법서만 처리하세요. 여러 파일을 동시에 업로드하면 혼동될 수 있습니다.
- 추출 결과가 기대와 다르면 구체적으로 어떤 부분이 틀렸는지 알려주면 Claude가 수정합니다.
- product_overrides.json에 새 상품의 override를 추가해야 하면, 파일을 수정 후 프로젝트에 다시 업로드하세요.
