# 세트데이터 추출 로직 흐름 문서

## 전체 파이프라인 개요

```
사업방법서 PDF ──┐
                ├──▶ [1] 상품분류 추출 ──▶ 상품분류/*.json
                │         (extract_product_classification.py)
                │
                ├──▶ [2] 세트데이터 추출 (4종)
                │    ├── 가입가능보기납기  (extract_insurance_period.py)
                │    ├── 납입주기          (extract_payment_cycle.py)
                │    ├── 보기개시나이      (extract_annuity_age.py)
                │    └── 가입가능나이      (extract_join_age.py)
                │
                └──▶ [3] 코드 매핑 ──▶ 코드매핑/{dataset}/*.json
                          (map_product_code.py)
                              │
                              ▼
                     [4] 정답 비교
                          (compare_product_data.py)
```

각 세트데이터 추출 스크립트는 **PDF(사업방법서)** 와 **JSON(상품분류)** 을 입력으로 받아,
상품별 세트데이터를 JSON으로 출력한다.

## 1. 가입가능보기납기 (`extract_insurance_period.py`)

보험기간 × 납입기간 조합과 제2보기개시나이를 추출한다.

### 1.1 전체 흐름

```
PDF + 상품분류 JSON
  │
  ├─[A] PDF 텍스트/테이블 추출 (extract_pdf_content)
  │     └── pdfplumber로 최대 30페이지 파싱
  │
  ├─[B] 보험기간 추출 (extract_insurance_periods)
  ├─[C] 납입기간 추출 (extract_payment_periods)
  │
  ├─[D] Context 기반 필터링 (3단계 우선순위)
  │     ├── 1순위: 테이블 기반 context map
  │     ├── 2순위: 텍스트 기반 context map
  │     └── 3순위: 섹션 기반 context map
  │
  ├─[E] 레코드 생성
  │     ├── 연금 상품: 연금개시나이별 레코드
  │     └── 비연금 상품: 보험기간 × 납입기간 직교곱
  │
  ├─[F] 레코드 필터링
  │     ├── N+X 조합 제외
  │     ├── 납입기간 ≤ 보험기간 제약
  │     └── 매트릭스 테이블 무효 조합 제외
  │
  └─[G] 오버라이드 적용 (product_overrides.json) → 최종 JSON 출력
```

### 1.2 보험기간 추출 (`extract_insurance_periods`)

PDF 텍스트에서 아래 패턴을 찾는다:

| 패턴 | 예시 | 추출 결과 |
|------|------|-----------|
| `n세만기` | 60세만기 | `60세` |
| `n년만기` | 20년만기 | `20년` |
| `종신만기` / `보험기간 : 종신` | - | `종신` |

**주의:** `종신납`, `연금`, `보험` 등 오인식 방지 키워드가 포함된 줄은 제외한다.

### 1.3 납입기간 추출 (`extract_payment_periods`)

| 패턴 | 예시 | 추출 결과 |
|------|------|-----------|
| `n년납` | 5년납 | `5년납` |
| `n~m년납` | 5~10년납 | `{5년납, 6년납, ..., 10년납}` |
| `n세납` | 20세납 | `20세납` |
| `일시납` | - | `일시납` (조건부 필터 적용) |
| `전기납` | - | `has_jeonginap = True` 플래그 설정 |
| `종신납` | - | `종신` |

**일시납 필터:** "전환형", "거치형", "즉시형", "계좌이체" 등의 맥락에서는 제외한다.

### 1.4 Context 기반 필터링

같은 PDF에 여러 상품 유형(1종/2종, 일반형/상생협력형 등)의 기간 정보가 섞여 있으므로,
상품명과 매칭되는 context의 기간만 필터링한다.

#### 1단계: 테이블 기반 (`_extract_context_period_map`)
PDF 테이블에서 fill-down 로직으로 컨텍스트와 기간값을 분리한다.

```
구분                 │ 납입기간
1종·일부지급형I      │ 5년납, 7년납
적립형 계약          │ 종신납
```
→ `{'1종·일부지급형I': {'5년납', '7년납'}, '적립형 계약': {'종신납'}}`

#### 2단계: 텍스트 기반 (`_extract_text_context_pay_map`)
`(1)`, `(가)` 등 번호 패턴 뒤의 컨텍스트와 `:` 뒤의 기간값을 파싱한다.

```
(1) 1종(출생아가입형) : 5년납, 7년납, 10년납
(가) 적립형 : 5~10년납, 15년납
```

#### 3단계: 섹션 기반 (`_extract_section_context_map`)
`(1)보험기간`, `(2)납입기간` 등 큰 섹션 아래에서 `(가)일반형`, `(나)상생협력형` 같은
하위 컨텍스트를 구분하여 보험기간/납입기간을 각각 수집한다.

```
(1) 보험기간
  (가) 일반형         →  ins: {90세, 100세}
  (나) 상생협력형     →  ins: {20년}
(2) 납입기간
  (가) 일반형         →  pay: {5년납, 7년납, ...}
  (나) 상생협력형     →  pay: {전기납}
```

**우선순위 규칙:**
- 일반적으로 테이블 기반 > 텍스트 기반 > 섹션 기반 순서로 적용
- 단, **일반형/상생협력형 분리**가 섹션 기반에서 감지되면, 테이블 기반보다 우선 적용
  (테이블이 두 유형을 혼용할 수 있기 때문)

#### 필터링 적용 (`_filter_periods_by_context`)
상품명의 토큰을 context key와 대조하여 매칭되는 context의 기간만 유지한다.

- 일반형/상생협력형 분리: 상품명에 "상생"이 포함되면 상생협력형 context만, 아니면 일반형 context만 사용
- 숫자 기간은 정확 매칭: "5년납" ≠ "15년납" (앞뒤 경계 검사)
- 매칭 결과가 없으면 원본 전체를 반환 (fallback)

#### 전기납 Context 필터
일반형/상생협력형이 분리된 경우, 매칭된 context에 `전기납`이 포함되지 않으면
`has_jeonginap` 플래그를 비활성화한다.

### 1.5 레코드 생성

#### 비연금 상품 (직교곱)
```
for ins in ins_periods:
    for pay in pay_periods:
        records.append(build_record(ins, pay))
    if has_jeonginap:
        records.append(build_record(ins, ins))   # 전기납: 납입기간 = 보험기간
```

#### 연금 상품
- **종신연금형:** `보험기간 = 종신`, 각 납입기간별 레코드 생성, 제2보기개시나이 포함
- **확정기간연금형:** `보험기간 = 연금개시나이 + 확정기간`, 나이별 개별 레코드 생성
- **전기납:** 연금개시나이별 `납입기간 = n세납` 레코드 생성

#### `build_record()` 출력 형식
```json
{
  "보험기간": "X90",        "보험기간구분코드": "X", "보험기간값": "90",
  "납입기간": "N20",        "납입기간구분코드": "N", "납입기간값": "20",
  "최소제2보기개시나이": "", "최대제2보기개시나이": "", "제2보기개시나이구분코드": ""
}
```

코드 변환 규칙:

| 원본 텍스트 | 코드 | 구분코드 | 값 |
|-------------|------|----------|-----|
| 종신 | A999 | A | 999 |
| 일시납 | N0 | N | 0 |
| 60세 / 60세납 | X60 | X | 60 |
| 20년 / 20년납 | N20 | N | 20 |

### 1.6 레코드 필터링

생성된 레코드에 아래 규칙을 순차적으로 적용한다:

| 필터 | 조건 | 예시 |
|------|------|------|
| N+X 제외 | 보험기간=N, 납입기간=X | N20 + X30 → 제거 |
| X<70 + N>20 제외 | 보험기간 X타입<70, 납입기간 N타입>20 | X60 + N30 → 제거 |
| 납입>보험 제외 | 같은 타입에서 납입기간값 > 보험기간값 | X90 + X100납 → 제거 |
| 매트릭스 무효 | PDF 테이블에서 `-` 표시된 조합 | (20년, 20년납) → 제거 |

#### 매트릭스 테이블 파싱 (`_extract_matrix_invalid_pairs`)
```
구분      20년만기  60세만기  70세만기  80세만기  100세만기
5년납      -        O        O        O        O
20년납     -        O        O        O        O
전기납     -        O        O        O        O
60세납     -        -        O        O        O
```
→ `{('20년', '5년납'), ('20년', '20년납'), ('60세', '60세납'), ...}` 무효 쌍 추출

**전기납 특수 처리:** `ins == pay`인 전기납 레코드는 매트릭스의 `전기납` 행만 참조한다.
(예: X60+X60은 전기납이므로 `60세납` 행이 아닌 `전기납` 행으로 판단)

## 2. 납입주기 (`extract_payment_cycle.py`)

보험료 납입 주기(월납, 연납, 일시납 등)를 추출한다.

### 2.1 전체 흐름

```
PDF + 상품분류 JSON
  │
  ├─[A] PDF 텍스트/테이블 추출
  │
  ├─[B] 납입주기 키워드 추출 (extract_cycle_names)
  │     └── 정규식으로 주기 키워드 탐색
  │
  ├─[C] 규칙 추출 (extract_cycle_rules)
  │     ├── 텍스트 기반 규칙 (parse_rules_from_text)
  │     └── 테이블 기반 규칙 (parse_rules_from_table)
  │
  ├─[D] 상품별 규칙 매칭 (enrich_records_with_cycles)
  │     ├── match_record_context() → 컨텍스트 매칭
  │     └── pick_cycles() → 최적 규칙 선택
  │
  └─[E] 오버라이드 적용 (product_overrides.json) → 최종 JSON 출력
```

### 2.2 납입주기 키워드 추출 (`extract_cycle_names`)

PDF 텍스트에서 아래 키워드를 탐색한다:

| 추출 패턴 | 정규화 결과 | 값 |
|-----------|-------------|-----|
| 일시납 | 일시납 | 0 |
| 월납 / 수시납 | 월납 | 1 |
| 3개월납 / 분기납 | 3개월납 | 3 |
| 6개월납 / 반기납 | 6개월납 | 6 |
| 연납 / 년납 | 연납 | 12 |

**주의:** `10년납`처럼 숫자가 앞에 붙은 `년납`은 납입주기가 아닌 납입기간이므로 제외한다.

### 2.3 규칙(CycleRule) 구조

```python
CycleRule(
    contexts=('2종', '적립형'),      # 매칭 조건 토큰들 (AND)
    cycles=('월납',),                # 해당하는 납입주기
    priority=2                       # = len(contexts), 구체적일수록 높음
)
```

#### 텍스트 기반 규칙 추출 (`parse_rules_from_text`)
```
(가) 보장형 계약, New Start 계약 : 월납
(나) 적립형 계약 : 수시납
```
→ `CycleRule(contexts=('보장형 계약',), cycles=('월납',), priority=1)`
→ `CycleRule(contexts=('적립형 계약',), cycles=('월납',), priority=1)`

조건부 규칙도 파싱:
```
월납(단, 3종의 경우 일시납)
```
→ 기본 규칙: `CycleRule(contexts=(), cycles=('월납',))`
→ 오버라이드: `CycleRule(contexts=('3종',), cycles=('일시납',), priority=1)`

#### 테이블 기반 규칙 추출 (`parse_rules_from_table`)
PDF 테이블에서 fill-forward(병합 셀 처리) 후 컨텍스트 열과 주기 열을 분리한다.

### 2.4 규칙 매칭

#### `match_record_context(record, rule)`
상품 레코드의 `상품명 + 세부종목` 텍스트에 rule의 모든 context 토큰이 포함되는지 확인한다.

```
record: 상품명="... 2종(계좌이체용) 적립형 종신연금형"
rule:   contexts=('2종', '적립형')
→ "2종" ✓, "적립형" ✓ → 매칭 성공
```

#### `pick_cycles(record, rules, fallback)`
1. 모든 규칙 중 매칭되는 것만 필터
2. `priority`가 가장 높은 규칙의 주기를 채택
3. 매칭 규칙이 없으면 `fallback_cycles` 사용
4. `CYCLE_ORDER = ['0', '1', '3', '6', '12']` 순서로 정렬

### 2.5 출력 형식

```json
{
  "상품명칭": "한화생명 ...",
  "상품명": "한화생명 ... 1종(출생아가입형)",
  "납입주기": [
    {"납입주기명": "월납", "납입주기값": "1", "납입주기구분코드": "M"},
    {"납입주기명": "연납", "납입주기값": "12", "납입주기구분코드": "M"}
  ]
}
```

## 3. 보기개시나이 (`extract_annuity_age.py`)

연금 상품의 제2보기개시나이(연금개시나이)를 추출한다.
연금 상품이 아닌 경우 빈 배열을 반환한다.

### 3.1 전체 흐름

```
PDF + 상품분류 JSON
  │
  ├─[A] PDF 텍스트/테이블 추출
  │
  ├─[B] Annuity Block 추출
  │     ├── 테이블 기반 (extract_annuity_blocks_from_tables)
  │     └── 텍스트 기반 (extract_annuity_blocks)
  │
  ├─[C] 체증경과년수 추출 (extract_escalation_pairs)
  │
  ├─[D] 상품별 매칭 (merge_records)
  │     ├── 연금 상품 → pick_annuity_values_for_row()
  │     └── 종신/체증형 → pick_escalation_for_row()
  │
  └─[E] 최종 JSON 출력
```

### 3.2 Annuity Block 구조

PDF에서 추출된 연금개시나이 정보를 담는 데이터 구조이다.

```python
{
    'generic_values': {         # 컨텍스트 무관 값
        '남자': [55, 56, ..., 80],
        '여자': [55, 56, ..., 80],
        '':     [45, 46, ..., 80],   # 성별 무관
    },
    'category_values': {        # 컨텍스트별 값
        '종신연금형': {
            '남자': [55, ..., 80],
            '여자': [55, ..., 80],
        },
        '확정기간연금형·1종': {
            '': [45, ..., 110],
        },
    },
    '_source': 'table',         # 테이블 추출 시 보너스 점수
}
```

#### 테이블 기반 추출
- 테이블 헤더에서 "종신연금형", "확정기간연금형" 등 카테고리를 감지
- 열별로 다른 카테고리의 나이 범위를 수집
- 성별(남자/여자) 행 구분

#### 텍스트 기반 추출
- `연금개시나이`, `제2보험기간 개시나이` 등 섹션 헤더 감지
- 나이 범위 패턴 파싱: `45세~80세`, `55세 이상 80세 이하`
- 컨텍스트 토큰(종신연금형, 1종 등) 추적

### 3.3 나이 범위 파싱 (`split_age_values`)

| 패턴 | 예시 | 결과 |
|------|------|------|
| 범위 | 45세~80세 | [45, 46, ..., 80] |
| 이상~이하 | 55세 이상 80세 이하 | [55, 56, ..., 80] |
| 쉼표 | 60세, 70세, 80세 | [60, 70, 80] |
| 단일값 | 55세 | [55] |

### 3.4 점수 기반 매칭 (`pick_annuity_values_for_row`)

상품 레코드에 가장 적합한 annuity block과 category를 선택하기 위해 다중 요소 점수를 사용한다.

| 우선순위 | 요소 | 설명 |
|----------|------|------|
| 1 | **정확 매칭** (0~2) | 상품 토큰 ⊆ category_key 토큰이면 2점 |
| 2 | **토큰 겹침** | 상품 토큰과 category_key 토큰의 교집합 수 |
| 3 | **소스 보너스** (0~1) | 테이블 추출이면 1점 (텍스트보다 정확) |
| 4 | **키 길이** | category_key 토큰 수 (타이브레이커) |

**신부부형 특수 처리:** 성별 분리값이 있는 매칭을 우선한다.

### 3.5 체증경과년수 (`extract_escalation_pairs`)

## 4. 가입가능나이 (`extract_join_age.py`)

성별/연령별 가입 가능 나이 범위를 추출한다. 가입가능보기납기 결과를 참조하여 보험기간/납입기간별 나이 범위를 생성한다.

### 4.1 전체 흐름

```
PDF + 상품분류 JSON + 가입가능보기납기 JSON
  │
  ├─[A] PDF 텍스트/테이블 추출 (extract_pdf_content)
  │     └── pdfplumber로 최대 30페이지 파싱
  │
  ├─[B] 가입나이 섹션 탐색 (_find_age_section)
  │     └── "보험기간 가입나이" / "가입나이" 헤더 위치 탐지
  │
  ├─[C] 전략 감지
  │     ├── detect_max_age_strategy() → 'simple' | 'formula' | 'table'
  │     └── detect_gender_split() → 성별 분리 여부
  │
  ├─[D] 3단계 추출 (merge_join_age_info)
  │     ├── Tier 1: 테이블 기반 (6가지 테이블 유형)
  │     ├── Tier 2: 수식 기반 (연금 상품)
  │     └── Tier 3: 텍스트 기반 (패턴 매칭)
  │
  ├─[E] 후처리
  │     ├── 상품별 오버라이드 (product_overrides.json)
  │     ├── 가입가능보기납기 교차 검증
  │     └── 실손의료비 특수 규칙 (product_overrides.json)
  │
  └─[F] 최종 JSON 출력
```

### 4.2 Tier 1: 테이블 기반 추출 (6가지 유형)

가장 정확한 추출 방식. PDF 테이블 구조를 인식하여 데이터를 파싱한다.

| 유형 | 함수 | 대상 | 테이블 형태 |
|------|------|------|-------------|
| 1a | `parse_age_table` | 일반 보험 | 보험기간×성별 매트릭스 |
| 1b | `parse_direct_age_table` | 즉시형 연금 | 구분 \| 남자 \| 여자 |
| 1c | `parse_inline_age_table` | 상해/골절보험 | 셀 내 "만N~M세" |
| 1d | `parse_variant_age_table` | 종별 분리 상품 | 1종 \| 2종 × 성별 |
| 1e | `parse_max_age_table` | 최고가입나이 상품 | 구분 \| 납입기간 \| 최고가입나이 |
| 1f | `parse_spin_paym_age_table` | 적립/거치형 연금 | 연금개시나이 × 납입기간 |

#### 1a 표준 매트릭스 테이블 (`parse_age_table`)
```
            │ 90세만기      │ 100세만기
            │ 남자   여자   │ 남자   여자
전기납      │ 27~75  45~75  │ 27~75  45~75
5년납       │ 15~80  15~80  │ 15~80  15~80
```
- split header 감지 (병합 헤더 행 처리)
- fill-right 로직으로 성별/보험기간 칼럼 매핑
- 다중 페이지 테이블 그룹 결합

#### 1f SPIN×PAYM 매트릭스 (`parse_spin_paym_age_table`)
```
연금개시나이 │ 거치형 │ 적립형(7년납, 10년납, 15년납, 20년납, 전기납)
45세        │ 44     │ 38   35   30   25   40
```
→ 거치형: MAX_AG = SPIN - deduction, 적립형: MAX_AG = SPIN - PAYM별 공제값

### 4.3 Tier 2: 수식 기반 추출 (`compute_formula_ages`)

연금 상품에서 PDF 텍스트에 공식이 기술된 경우:

**공식:** `최대가입나이 = 연금개시나이 - 납입기간 - 공제값`

- **공제 테이블** (`parse_deduction_table`):
  ```
  N년납 (연금개시나이 – D)세  →  paym_deductions[N년납] = D
  전기납 (연금개시나이 – D)세  →  paym_deductions[전기납] = D
  ```
- **거치형 모드:** `MAX_AG = SPIN - deduction` (PAYM 없음)
- **적립형 모드:** (SPIN, PAYM) 조합별 계산

### 4.4 Tier 3: 텍스트 기반 추출

테이블도 공식도 없는 경우, 가입나이 섹션 내에서 "N~M세" 패턴을 탐색한다.
상품명 context로 매칭되는 줄만 사용.

### 4.5 후처리

#### 가입가능보기납기 교차 검증
추출된 나이 레코드의 보험기간/납입기간이 가입가능보기납기 결과에 존재하는지 검증.
미매칭 조합은 제거한다.

#### 실손의료비 특수 규칙
| 조건 | min_age | max_age |
|------|---------|---------|
| 태아가입용 | 0 | 0 |
| e실손 | 19 | (추출값) |
| 간편가입+재가입 | 8 | 109 |
| N세형 | (추출값) | N-1 |
| 종신갱신형 | (추출값) | 199 |

#### 연금 Sibling Fallback
거치형 연금이 자체 SPIN 데이터 없을 시, 같은 상품명칭의 형제 상품(적립형)에서 SPIN 데이터 차용.

### 4.6 출력 형식

```json
{
  "상품명칭": "한화생명 ...",
  "상품명": "한화생명 ... 1종(출생아가입형)",
  "가입가능나이": [
    {
      "성별": "1",                          // 1=남자, 2=여자, ""=공통
      "최소가입나이": "15",
      "최대가입나이": "80",
      "최소납입기간": "5",  "최대납입기간": "5",  "납입기간구분코드": "N",
      "최소제2보기개시나이": "", "최대제2보기개시나이": "", "제2보기개시나이구분코드": "",
      "최소보험기간": "90", "최대보험기간": "90", "보험기간구분코드": "X"
    }
```

## 5. 코드 매핑 (`map_product_code.py`)

추출 결과를 보종코드/상품코드 CSV 매핑 테이블과 연결한다.

### 5.1 전체 흐름

```
추출 결과 JSON + 보종코드_상품코드_매핑.csv
  │
  ├─[A] CSV 매핑 테이블 로드 (261건)
  │     └── ISRN_KIND_DTCD, ISRN_KIND_ITCD, PROD_DTCD 등
  │
  ├─[B] 상품명 매칭 (match_codes)
  │     └── 4단계 fallback 매칭
  │
  ├─[C] Sibling Fallback
  │     └── 미매핑 CSV 행에 형제 데이터 복사
  │
  └─[D] 최종 JSON 출력 (코드 + 추출 데이터)
```

### 5.2 4단계 Fallback 매칭 (`match_codes`)

| 단계 | 매칭 대상 | 방식 |
|------|-----------|------|
| 1 | `prod_sale_nm` | 토큰 집합 포함 (가장 엄격) |
| 2 | `prod_sale_nm` | 부분문자열 매칭 (경계 인식) |
| 3 | `isrn_kind_sale_nm` | 토큰 집합 포함 |
| 4 | `isrn_kind_sale_nm` | 부분문자열 매칭 (경계 인식) |

#### 추가 필터
- **납입면제형:** 상품에 포함 시 매핑 행에도 있어야 함 (역도 동일)
- **간편가입:** 동일 로직
- **일부지급형 레벨:** I/II/III 로마 숫자 정확 매칭

#### 최종 순위
1. 여분 토큰 수 최소화 (과매칭 방지)
2. 대괄호 수 최소화
3. 필수 토큰 겹침 최대화

### 5.3 Sibling Fallback

매핑되지 못한 CSV 행 중, 같은 `(DTCD, ITCD)` 쌍에 매핑된 형제가 있으면
형제의 추출 데이터를 복사한다.

```
CSV 행 A: DTCD=2126, ITCD=A01 → 매핑됨 (추출 데이터 있음)
CSV 행 B: DTCD=2126, ITCD=A01 → 미매핑
→ A의 추출 데이터를 B에 복사
```

## 6. 정답 비교 (`compare_product_data.py`)

코드 매핑 결과를 정답 CSV와 비교하여 정확도를 산출한다.

### 6.1 비교 흐름

```
코드매핑 결과 JSON + 정답 CSV + 매핑 CSV
  │
  ├─[A] 정답 CSV 로드 (key: DTCD + ITCD + SALE_NM)
  ├─[B] 매핑 결과 JSON 로드
  │
  ├─[C] 데이터셋별 비교 함수 실행
  │     ├── insurance_period: 보험기간/납입기간 조합 세트 비교
  │     ├── payment_cycle: 납입주기값 세트 비교
  │     ├── annuity_age: 보기개시나이 범위 비교 (성별 고려)
  │     └── join_age: 가입가능나이 범위 비교 (성별/기간별)
  │
  └─[D] 리포트 생성
        ├── Extraction-based: 추출 행 기준 정확도
        └── CSV-row-based: CSV 행 기준 정확도 (최종 지표)
```

### 6.2 비교 기준

| 데이터셋 | 비교 대상 | 비교 방식 |
|----------|-----------|-----------|
| 가입가능보기납기 | `{(보험기간, 납입기간, 제2보기개시나이)}` 집합 | 양방향 집합 일치 |
| 납입주기 | `{납입주기값}` 집합 | 양방향 집합 일치 |
| 보기개시나이 | `{(제2보기개시나이, 구분코드)}` 리스트 | 성별별 범위 비교 |
| 가입가능나이 | `{(성별, min, max, INS, PAYM, SPIN)}` 집합 | 다차원 튜플 집합 비교 |
